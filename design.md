# 个人产品管理系统 — 产品设计文档

> 版本：v1.0 | 日期：2026-06-16 | 状态：设计阶段

---

## 1. 产品概述

### 1.1 产品定位

一个轻量级的个人产品/项目管理工具，用于集中展示和管理自己创建的小项目。用户可以登录后在管理界面增删改自己的产品信息，主页则以卡片形式公开展示所有已上线的产品。

### 1.2 目标用户

- **主要用户**：系统拥有者本人。以单用户为主，但采用基于角色的双账号体系
- **两种角色**：
  - `admin`（管理员）：可访问管理页，进行产品的增删改
  - `user`（普通用户）：可登录并浏览，但无管理页访问权限（删除产品仅管理员可执行）
- 无需注册功能，账号密码硬编码

### 1.3 核心价值

- 统一展示个人所有产品/项目，方便对外分享
- 支持动态管理（增删改），无需改代码
- 从 0 到 1 搭建，适合学习 Vue 3 前端工程化全流程
- 纯静态站点，可免费部署到 GitHub Pages

## 2. 功能模块

### 2.1 登录模块

- 登录页面，用户名 + 密码表单
- 内置两个账号（前后端一致）：`admin`/`admin123`（管理员）、`user`/`user123`（普通用户）
- 登录成功后存储 token 与用户信息（含角色）到 `localStorage`，前端路由守卫校验登录状态与角色
- 登录优先调用后端 `/api/login`，后端不可达时降级为前端本地账号表校验

### 2.2 主页（产品展示）

- 以卡片网格布局展示所有 **状态为"上线"** 的产品
- 每张卡片展示：图标、产品名称、简介、跳转链接按钮
- 右上角显示用户头像 + 下拉菜单（退出登录）
- 支持响应式布局（PC 端 3 列，平板 2 列，移动端 1 列）
- 返回顶部按钮（页面滚动超过一屏时显示）
- 按优先级从高到低排序（priority 数值越小越靠前）

### 2.3 管理页面（产品管理）

- 登录后才能访问（路由守卫保护）
- 以表格形式展示所有产品（含"开发中"状态的产品）
- 支持搜索过滤（按产品名称模糊匹配）
- 支持操作：
  - **新增产品**：弹出表单填写产品信息
  - **编辑产品**：弹出表单修改已有产品信息
  - **删除产品**：确认后删除
  - **切换状态**：快速切换上线/开发中
- 数据操作当前在前端内存中完成（Pinia store），后续可切换为调用后端 API

### 2.4 导航与布局

- 侧边导航栏暂时隐藏，但代码结构预留侧边栏组件和插槽
- 页面间跳转通过顶栏或页面内链接实现（首页 ↔ 管理页）
- 布局组件统一管理顶栏、内容区、底栏

## 3. 数据模型

### 3.1 产品实体（Product）

| 字段       | 类型      | 必填 | 说明                                   |
| ---------- | --------- | ---- | -------------------------------------- |
| id         | string    | 是   | 唯一标识，UUID 生成                    |
| name       | string    | 是   | 产品名称，最长 50 字符                 |
| description| string    | 否   | 产品简介，最长 200 字符                |
| url        | string    | 是   | 跳转链接（完整 URL）                   |
| icon       | string    | 否   | 图标 URL 或 emoji，默认使用 🔗         |
| tags       | string[]  | 否   | 标签列表，如 ["Vue", "工具"]           |
| status     | enum      | 是   | `online`（上线） / `developing`（开发中）|
| priority   | number    | 是   | 排序优先级，数值越小越靠前，默认 99    |
| createdAt  | string    | 是   | 创建时间，ISO 8601 格式                |
| updatedAt  | string    | 是   | 更新时间，ISO 8601 格式                |

### 3.2 用户实体（User）

| 字段     | 类型   | 说明                          |
| -------- | ------ | ----------------------------- |
| username | string | 用户名（硬编码）              |
| password | string | 密码（硬编码）                |
| role     | enum   | `admin`（管理员）/ `user`（普通用户） |
| avatar   | string | 头像 URL                      |
| nickname | string | 显示昵称                      |

## 4. 页面路由设计

| 路由路径      | 页面名称 | 权限     | 说明                     |
| ------------- | -------- | -------- | ------------------------ |
| `/login`      | 登录页   | 公开     | 未登录自动重定向到此     |
| `/`           | 主页     | 公开     | 产品展示，默认首页       |
| `/manage`     | 管理页   | 需登录   | 产品增删改查             |

> 路由守卫逻辑：
> - 访问 `/manage` 时，检查 `localStorage` 中是否有 token，无则跳转 `/login`
> - 已登录用户访问 `/login` 时，自动跳转 `/`

## 5. API 接口

前后端分离架构已落地。后端（Spring Boot）已实现以下接口，前端通过 `api/` 目录下的 fetch 客户端调用；当后端不可达时自动降级为前端本地数据（详见 6.4 优雅降级）。

| 接口        | 方法   | 说明         | 权限         | 当前实现                       |
| ----------- | ------ | ------------ | ------------ | ------------------------------ |
| `/api/login`| POST   | 用户登录     | 公开         | 内存用户表校验，返回 mock token |
| `/api/products` | GET | 获取产品列表 | 公开         | 返回内存存储中的数据            |
| `/api/products` | POST | 新增产品     | 需登录       | 写入内存存储                    |
| `/api/products/:id` | PUT | 编辑产品 | 需登录       | 更新内存存储中对应项（仅非空字段） |
| `/api/products/:id` | DELETE | 删除产品 | 需管理员     | 从内存存储中移除                |

> 后端统一响应包装为 `ApiResponse<T>`：`{ code, message, data }`，前端读取 `data` 字段。

## 6. 技术架构

### 6.1 前端技术栈

| 技术         | 选型              | 说明                               |
| ------------ | ----------------- | ---------------------------------- |
| 框架         | Vue 3             | Composition API + `<script setup>` |
| 构建工具     | Vite              | 快速开发与生产构建                  |
| 路由         | Vue Router 4      | SPA 页面路由                       |
| 状态管理     | Pinia             | 轻量级状态管理，管理产品数据和用户状态 |
| 样式方案     | 原生 CSS + Element Plus | CSS 变量管理设计 token，UI 组件用 Element Plus |
| 图标         | Element Plus Icons | 配合 Element Plus 使用             |
| 表单验证     | 原生逻辑          | 暂不引入第三方验证库               |
| HTTP 请求    | 原生 fetch        | `api/index.js` 手写封装，零外部依赖 |

### 6.2 后端技术栈

当前后端已实现基础 CRUD 与认证，使用**内存存储**（`ConcurrentHashMap`，重启即丢）。数据库、Spring Security、JWT 依赖已在 `pom.xml` 中预置但暂时注释，待持久化阶段启用。

| 技术       | 选型                | 状态           |
| ---------- | ------------------- | -------------- |
| 语言       | Java 17             | 已使用         |
| 框架       | Spring Boot 3.2     | 已使用         |
| 构建工具   | Maven               | 已使用         |
| 数据存储   | 内存 `ConcurrentHashMap` | 当前实现   |
| 认证       | Mock token（内存映射）   | 当前实现，JWT 预留 |
| 数据库     | MySQL 8.0 / H2      | 预留（依赖已注释） |
| ORM        | MyBatis-Plus        | 预留（依赖已注释） |

## 7. UI/UX 设计规范

### 7.1 色彩系统

| 用途            | 色值     | 说明               |
| --------------- | -------- | ------------------ |
| 主色（Primary） | `#7C5CFC`| 浅紫色，按钮、链接 |
| 主色浅          | `#A78BFA`| 悬浮态、次要元素   |
| 主色深          | `#5B3CC4`| 按下态、强调文字   |
| 背景色          | `#F8F7FF`| 页面底色，微紫调   |
| 卡片/表背景     | `#FFFFFF`| 纯白               |
| 主文字色        | `#1E1B4B`| 深紫黑             |
| 次要文字色      | `#6B7280`| 灰色               |
| 边框色          | `#E5E7EB`| 浅灰               |
| 成功/上线       | `#10B981`| 绿色               |
| 警告/开发中     | `#F59E0B`| 橙黄色             |
| 危险/删除       | `#EF4444`| 红色               |

### 7.2 字体与排版

- 系统字体栈：`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, "Noto Sans SC", sans-serif`
- 标题字号：28px / 22px / 18px
- 正文字号：16px
- 辅助文字：14px
- 行高：1.6

### 7.3 响应式断点

| 断点     | 宽度        | 卡片列数 | 说明     |
| -------- | ----------- | -------- | -------- |
| 移动端   | < 640px     | 1 列     | 堆叠布局 |
| 平板     | 640-1024px  | 2 列     | 紧凑布局 |
| PC 端    | > 1024px    | 3 列     | 完整布局 |

### 7.4 关键交互

- **产品卡片悬浮**：轻微上浮 + 阴影加深（`transform: translateY(-4px)`）
- **按钮悬浮**：颜色变浅（主色 → 主色浅）
- **模态弹窗**：居中弹出，背景半透明遮罩，点击遮罩可关闭
- **返回顶部**：右下角固定圆形按钮，滚动超过 300px 时渐显
